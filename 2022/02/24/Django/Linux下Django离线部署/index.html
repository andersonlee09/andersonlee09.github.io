Linux下Django离线部署

Linux下Django离线部署

在这里介绍一下Linux下Django离线部署的过程，大概分为以下三步

  1. 首先就是python manage.py runserver
  2. 第二步就是 使用uwsgi进行代理
  3. nginx反向代理，这一步可以降低服务器对于静态资源的应对能力，实现动静分离，增加效率。

1.首先关闭防火墙

systemctl disable firewalld

chkconfig iptables off

2.关闭selinux

vi /etc/selinux/config

3.ping www.baidu.com

4.用yum安装包

yum -y install wget vim

wget https://www.python.org/ftp/python/3.7.6/Python-3.7.6.tgz

安装opt下

安装python依赖

yum -y install zlib-devel bzip2-devel openssl-devel ncurses-devel sqlite-devel readline-devel tk-devel gdbm-devel db4-devel libpcap-devel xz-devel

解压python

tar -xzvf Python-3.7.6.tgz -C /usr/local/src/

cd /usr/local/src/

cd Python-3.7.6/

./configure –prefix=/usr/local/python3

make && make install

添加环境变量

cd /etc/profile.d/

vi python3.sh

添加export PATH=”$PATH:/usr/local/python3/bin”

重载环境变量 source ../profile

echo $PATH

############33

实际部署

cd /opt

mkdir wxapp

cd wxapp

mkdir script

上传项目至wxapp

pip3 install –upgrade pip

pip freeze >requirements.txt

pip3 download -d packages -r requirements.txt

修改ALLOW_HOST

pip3 install uwsgi

这一步一定是再在wweixinapp目录下

uwsgi –http 0.0.0.0:8000 –file weixinapp/wsgi.py –static-map=/static=static

cd ../script

vim uwsgi.ini

[uwsgi]
# 项目目录
http=0.0.0.0:8000
chdir=/opt/wxapp/weixinapp/
# 启动uwsgi的用户名和用户组
uid=root
gid=root
# 指定项目的application
module=weixinapp.wsgi:application
# 指定sock的文件路径
socket=/opt/wxapp/script/uwsgi.sock
# 启用主进程
master=true
# 进程个数
workers=5
pidfile=/opt/wxapp/script/uwsgi.pid
# 自动移除unix Socket和pid文件当服务停止的时候
vacuum=true
# 序列化接受的内容，如果可能的话
thunder-lock=true
# 启用线程
enable-threads=true
# 设置自中断时间
harakiri=30
# 设置缓冲
post-buffering=4096
# 设置日志目录
daemonize=/opt/wxapp/script/uwsgi.log

uwsgi –ini uwsgi.ini # 启动uwsgi

uwsgi –stop uwsgi.pid

ps -ef | grep -i uwsgi

添加nginx配置文件

cd /www/server/nginx/conf/

vim weixinapp.conf

/etc/init.d/nginx start

/etc/init.d/nginx stop
