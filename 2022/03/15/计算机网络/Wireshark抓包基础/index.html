如图，左侧为自定义的名称，右侧为一些过滤条件，关于过滤条件我会细讲

过滤

表达式组合

针对ip

  • 对源ip地址过滤

ip.src == 192.168.0.1

  • 对目的地址过滤

ip.dst == 192.168.0.1

  • 对源地址或者目的地址进行过滤

ip.addr == 192.168.0.1

  • 如果想排除以上的数据包，只需要将其用括号囊括，然后使用 “!” 即可

!(ip.addr == 192.168.0.1)

针对协议

注意！！！！！这里的协议都是小写，不要用大写

  • 获某种协议的数据包，表达式很简单仅仅需要把协议的名字输入即可

http

  • 捕获多种协议的数据包

http or telnet

  • 排除某种协议的数据包

not arp 或者 !tcp

针对端口

  • 捕获某一端口的数据包（以tcp协议为例）

tcp.port == 80

  • 捕获多端口的数据包，可以使用and来连接，下面是捕获高于某端口的表达式（以udp协议为例）

udp.port >= 2048

针对长度和内容的过滤

针对长度的过虑（这里的长度指定的是数据段的长度）

udp.length < 20

http.content_length <=30

针对url的过滤

http.request.uri matches “user” (请求的uri中包含“user”关键字的)

针对http请求的一些过滤实例

  • 过滤出请求地址中包含“user”的请求，不包括域名；

http.request.uri contains “User”

  • 精确过滤域名

http.host==baidu.com

  • 模糊过滤域名

http.host contains “baidu”

  • 过滤请求的content_type类型

http.content_type ==”text/html”

  • 过滤http请求方法

http.request.method==”POST”

  • 过滤tcp端口

http && tcp.port==80 or tcp.port==5566

  • 过滤http响应状态码

http.response.code==302

  • 过滤含有指定cookie的http数据包

http.cookie contains “userid”
